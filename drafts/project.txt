Сайт, на котором пользователь может заказать и сконфигурировать перевозку своего груза в контейнере из нескольких портов Китая до России

1. Пользователь сможет выбрать порт отправления и необходимые услуги (фрахт, экспедирование, таможенное оформление и мб что-то еще)

2. Администратор сможет видеть направленные заявки и изменять в них данные


Страницы сайта:

КОММЕНТАРИЙ: "о компании" и "краткое описание деятельности" — звучит как почти одно и то же — я бы поместил этот текст в основной блок страницы; а ставки, услуги, регистрацию и всё остальное вынес бы в верхнее меню; но это вопрос дизайна, так что вы вольны сделать как угодно
- приветственная: страница с кратким описанием деятельности, с кнопками (каждая ведет на отдельную страницу:
    - о компании
    - ставки и услуги
    - заказать услугу (конфигуратор)
    - у менеджера сайта будет доп страница с готовыми заявками (пока не знаю куда ее прилепить)
    - регистрация

- заказать услугу
    - в конфигураторе на выбор будет 2 перевозчика, у которых будут отличаться ставки
      после выбора перевозчика будет одинаковый для обоих интерфейс с полями:
        - тип контейнера (20' 40' или реф), к рефу будет надбавка цены
        - указание веса груза (ограничение до 20 тонн)
        - указание названия груза
        - выбор порта отправления (от выбора порта цена зависеть не будет, только от перевозчика и типа контейнера)
      порт доставки будет один


ИСПРАВИТЬ: модель
    - для пользователя будет использоваться встроенный класс модели (таблица в БД), об этом я ещё буду говорить на лекциях
    - если Application предназначен для заявок, то его надо переименовать соответственно — а внешние ключи обозначены разумно
    - что касается цен, то, если я правильно понимаю, для веса цена вычисляется, а для дополнительных услуг цены являются фиксированными:
        - в таком случае вам не нужны отдельные классы (таблицы) для хранения цен, а нужны дополнительные атрибуты (поля) в классе (таблице) для заявок — см. пример ниже
        - вычисление полной стоимости заявки можно осуществлять специальным методом в классе модели заявок

ИСПОЛЬЗОВАТЬ: возможный вид таблицы для заявок
 ---------------------
 | orders            |
 ---------------------
 | id                |
 | user_id           |
 | name              |
 | weight            |
 | port_id           |
 | carrier_id        |
 | container_id      |
 | forwarding        |
 | customs_clearance |
 | total_price       |
 ---------------------
    для полей forwarding и customs_clearance в БД можно выбрать тип ENUM, записав в них несколько значений, соответствующих названиям видов дополнительных услуг за 200, 400 usd, ... и так далее
        https://dev.mysql.com/doc/refman/8.0/en/enum.html
    в классе модели этому типу будет соответствовать поле модели CharField с вариантами выбора:
        https://docs.djangoproject.com/en/4.1/ref/models/fields/#enumeration-types
